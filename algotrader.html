How it works:

Dashboard

We connect with the Binance client to obtain live cryptocurrency data, we pass this data as a JSON to a Lightweight Financial Chart. This displays a live chart of cryptocurrency data.

From the Binance client we check our cryptocurrency wallet's balance, and identify all currencies that have a positive balance and display them to the user.

Using the balances we obtain from the Binance client, we can find the minutely trading price for that cryptocurrency and calculate its value in dollars. We sum this value and display it to the user, write the data to a .csv file and upload it to a lightweight chart to display the change in the value of the portfolio over time.

Trading Algorithm

Data is received using Binance's webscocket client. This gives us a stream of candlestick data. Once the daily candle has closed, we write the Time, Open, High, Low, Close and Volume of the candle to a .csv file.

With our candlestick data, we use the Python BTA-Lib library to calculate the 'fast' moving average (12 days), the 'slow' moving average (26 days) and the Stochastic Indicator (12 days). When we detect a crossover - when the 'fast' moving average either goes above or below the 'slow' moving average we will check to buy or sell (respectively) the cryptocurrency. We verify our signals using the Stochastic Indicator, checking if the cryptocurrency is overbough or oversold. The Stochastic Indicator is a measure betweeon 0 and 100, typically overbought signals are when the indicator rises above 80 and oversold when it falls below 20. In this case, we are simply checking if the currency is overbought or oversold relative to 50. This encourages more trades to occur and gives more weight to the moving average crossover heuristic.

This algorithm has been tested on Cryptocurrency data over the past 3 years using the Python Backtrader module. From my technical analysis, it performs well on currencies that experience mid-term volatility, particularly bitcoin and ether.
